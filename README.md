# ai-design-rules

将 UI 组件库的设计规范体系提炼成 **AI 可执行的设计约束指令**，让 AI 生成接近设计师水准的 UI。

## 为什么需要这个

AI 生成 UI 质量低的根本原因不是"不懂代码"，而是缺乏系统性设计约束：

- 组件选型靠猜（什么时候用 Select，什么时候用 Radio？）
- 各页面视觉不一致（间距、颜色、对齐各自为政）
- 没有整体交互视角（先写组件，后发现流程对不上）
- 状态设计不完整（没有空态、错误态、权限态）

这套规则通过 Cursor Rules + Skills 解决以上问题。

## 内容架构

```
Level 1  设计哲学    让 AI 理解价值观，激活正确的设计认知
Level 2  设计原则    视觉四原则 + 交互六原则，让 AI 学会判断
Level 3  全局规范    字体/色彩/间距/阴影/动效的精确数值系统
Level 4  设计模式    数据录入、数据展示、导航、反馈的场景决策树
Level 5  组件规范    组件选择决策树，消除 AI 的选型模糊性
Skills   方法论      交互设计方法论 / 页面模板 / UI 审查
```
